Build a polished, single-file Streamlit app called "Tweet Analytics Dashboard".

**Context:** I am a social media manager analyzing a CSV of tweet exports. I need engagement charts plus OpenAI analysis of the account's vibe, personality, and a draft tweet in their voice.

**Tech Stack:**
- Streamlit (UI)
- Pandas (data)
- Altair (time-series scatter charts)
- Plotly (radar chart)
- OpenAI Python SDK (LLM calls)
- python-dotenv (API key from `.env`)

**Model:** Use **`gpt-5.6`** for every OpenAI call (vibe report, personality JSON, tweet generation).

**Design:**
- `layout="wide"`, clean dark-sidebar / light-main look inspired by X (not purple gradients).
- Soft metric cards, clear typography, primary accent `#1d9bf0`.
- Keep AI results in `st.session_state` so switching sections does not wipe them.

**Data:**
- Accept CSV upload with at least: `text`, `view_count`, `created_at`, `favorite_count`.
- Optionally use `retweet_count` / `reply_count` if present.
- Parse dates; coerce numerics; compute `engagement = favorite_count / view_count` (safe divide-by-zero).
- Sidebar: file uploader **plus** one-click demo loaders for sample CSVs already in the project folder.

**Sections (sidebar nav or tabs):**

1. **Overview** — metrics (tweets, favorites, avg favorites, max favorites, avg engagement %). Sortable tweet table.
2. **Engagement** — Altair scatters: date vs engagement; date vs favorites. Tooltips with tweet text.
3. **Vibe Report** — button runs GPT on a mix of top-engagement + recent tweets (~40). Return **raw HTML only** covering Persona, Writing Style, Engagement Insights. Render with `unsafe_allow_html`.
4. **Personality** — GPT with `response_format={"type": "json_object"}` for Big Five (+ a short summary). Plotly radar 0–100.
5. **Generate Tweet** — few-shot from top tweets; return tweet text only. Show an X-style preview card (avatar, handle, fake metrics).

**Quality bar:**
- Graceful empty state before upload.
- Clear spinner copy while models run.
- Strip markdown fences if the model wraps HTML/JSON by mistake.
- Do not hardcode API keys; read `OPENAI_API_KEY` from the environment.
