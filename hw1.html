Homework 1: Multimodal Insight Observer (React + Vite)

Due: Friday, July 24, 2026 at 11:59 PM

Build the Insight Observer as a React + Vite web app (not Streamlit). An AI watches a user react to a YouTube video through webcam frames, interviews them, and writes a final sentiment report. Submit a public GitHub repository structured so an automated grader can find your code and required outputs.

Setup help: see the course Node.js (for React Apps) and GitHub instructions. A simple webcam + OpenAI starter is at webcam_analysis (you may use ideas from it, but this homework must implement the full Insight Observer features below).

App Requirements

The user watches a video while the AI “watches” them. After the video, the AI runs a live interview, then produces a final report. Your app must include:

  1. YouTube Video Metadata: An input for a YouTube URL. When a URL is provided, programmatically retrieve the Video Title, Duration (in seconds), Description, and Transcript.
  2. Visual Evaluation: Use the webcam to capture reactions while the user watches.
    • Model: Use const MODEL = 'gpt-5.6-luna' for all AI processing.
    • Sampling: Capture and send a maximum of 20 images to the AI for the visual evaluation. Display the evaluation in a clear, readable layout.
  3. The Interviewer (Chatbot): After the video ends, a Start Interview button opens a chatbot. The AI’s system prompt must include the video metadata and visual evaluation. The AI should ask what the user liked/disliked and reference facial expressions (e.g., “I noticed you smiled at [Time X], what caused that?”).
  4. Final Synthesis: An End Chat button sends the chat history, video metadata, and visual evaluation to the AI for a final written summary of how the user felt. Display the report in a nicely formatted way.

Stack Requirements

  • Create the project with Vite’s React template, for example: 
    npm create vite@latest insight-observer -- --template react
cd insight-observer
npm install
npm run dev
  • Main UI in React (e.g. src/App.jsx or src/App.tsx).
  • Include a root package.json so the app installs with npm install and runs with npm run dev (usually at http://localhost:5173).
  • Do not commit API keys. Use a local .env (listed in .gitignore) with a Vite-style key such as VITE_OPENAI_API_KEY, and document it in your README.

Submission: GitHub Repository

Push your project to a GitHub repository and submit the repository URL on Canvas. The repo must include an ai_grading/ folder at the repository root with the exact outputs below so an AI grader can score your work without digging through the whole tree.

Required layout (folder names and filenames must match):

your-repo/ ├── package.json ├── vite.config.js (or vite.config.ts) ├── index.html ├── README.md (how to install, set env vars, and run) ├── .gitignore (must include .env and node_modules) ├── src/ │ ├── App.jsx (or App.tsx — main app) │ └── … (components, styles, helpers as needed) └── ai_grading/ ├── final_prompt.txt (exact final synthesis prompt from your test run) ├── visual_evaluation.txt ├── video_metadata.json └── final_report.txt (final synthesis report text from your test run)

What goes in ai_grading/

  • final_prompt.txt — the exact prompt sent for the Final Synthesis on the test video (must include YouTube metadata, visual evaluation, and chat history).
  • visual_evaluation.txt — the visual evaluation text produced for that same test run.
  • video_metadata.json — JSON with at least title, duration_seconds, description, and transcript for the test video.
  • final_report.txt — the final report text shown in the app after End Chat.

Do not commit webcam images, large media files, node_modules/, or secrets. Keep the grading folder text/JSON only. Do not submit a Streamlit app.py zip.

Interview Logic 20 Start Interview launches a chatbot whose system prompt includes the visual evaluation and video metadata.
Final Synthesis 20 End Chat produces a coherent report integrating chat history with visual/video data, shown in a clear layout.
AI Grading Folder 15 ai_grading/ exists at the repo root with final_prompt.txt, visual_evaluation.txt, video_metadata.json, and final_report.txt from a valid test-video run.
